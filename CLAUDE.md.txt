## CE QU'ON CONSTRUIT
Un outil interne de base de données candidats — un "moteur de recall" personnel, pas un CRM/ATS. Fichier HTML unique déployé sur **earlybird-talent.vercel.app** via GitHub + Vercel.

### Use case principal
Un client me donne les specs d'un job → je tape en langage naturel dans l'outil → il me sort les candidats pertinents en Tier 0/1/2/3 → je partage mon écran avec le client.

### 3 fonctions distinctes
1. **Base marché** — tous les candidats sourcés, enrichis automatiquement avec MES règles de classification
2. **Mémoire augmentée** — candidats avec qui j'ai parlé, avec mon jugement qualitatif (notes Notion)
3. **Base de boîtes** — entreprises classifiées par industrie/nature/taille (386 boîtes déjà faites)

## ARCHITECTURE & STACK
- **Frontend** : fichier HTML unique avec JS embarqué, window.storage pour persistance locale
- **Hosting** : Vercel (auto-deploy depuis GitHub)
- **APIs proxy** : `/api/notion.js` et `/api/claude.js` en serverless Vercel (tokens en env vars)
- **Notion** : outil de prise de notes pendant les calls, sync one-way vers l'app. DB "Candidate Calls" avec champs : Nom, LinkedIn URL, Projet/Client, Date du Call, Statut, Ranking
- **PhantomBuster** : enrichissement LinkedIn (76 profils/jour, en cours sur ~4400 restants). Fournit : jobTitle, company, jobLocation, headline, allSkills, description, school, etc.
- **Mot de passe app** : `Nhresr892701?` (sessionStorage)
- **Déploiement** : `deploy-earlybird.bat` sur le Bureau (double-clic → push Git)
- **Node.js** : installé (v24.14.1)

### Séniorité (identique toutes familles, calculée sur les années DANS le rôle exact)
- **Junior** : 0-2 ans
- **Mid** : 2-5 ans
- **Senior** : 5 ans+

### Règle clé
L'expérience se compte dans la famille de rôle, pas en expérience totale. 2 ans SDR + 2 ans AE = junior AE. Le matching de titres est intelligent (fuzzy, FR/EN, abréviations).

### Logique de search NL
- **Filtres éliminatoires** (pas de match = absent) : Rôle, Localisation
- **Critères de tiering** (souples, font varier le tier) : Séniorité, Industrie (à venir), Salaire (à venir), Cycle de vente (à venir)
- Tier 0 = tous critères souples OK / Tier 1 = 1 manquant / Tier 2 = 2 manquants / Tier 3 = 3 manquants
